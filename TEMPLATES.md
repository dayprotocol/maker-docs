Good default when you want a real book without heavy directional bias.\n\n**Best for:** Default book on any mint after warmup — steady maker presence on Meteora DLMM / Raydium / Orca while routing fills via Jupiter\n\n**How it works:** market_make-heavy weights; neutral inventory_skew; soft/hard inventory caps; stop_loss + daily_loss kill switch. Light sell_into_flow to unload into organic buyers. Pools: create/manage on Meteora DLMM or Raydium CLMM; swaps via Jupiter.\n\n**Venues:** Meteora DLMM, Raydium CLMM / AMMv4, Orca Whirlpools, Jupiter (swap routing)\n\n**Key params:** `base_spread_bps`, `max_inventory_sol`, `inventory_hard_cap_sol`, `stop_loss_pct`, `inventory_hard_cap_sol`, `inventory_soft_target_sol`\n\n### `tight_liquid_pmm` — Tight PMM Liquid\n\nHigh-liquidity pairs: narrow spreads, high refresh rate\n\n**Best for:** Deep books where tight quotes still fill; maximize maker turnover\n\n**How it works:** Narrow spreads, fast quote_refresh_ms, lower inventory, mild TP. Dynamic size reduced when vol spikes.\n\n**Venues:** Raydium CLMM / AMMv4, Orca Whirlpools, Meteora DLMM, Jupiter (swap routing)\n\n**Key params:** `base_spread_bps`, `quote_refresh_ms`, `dynamic_size_vol_factor`, `max_inventory_sol`\n\n### `wide_vol_pmm` — Wide PMM Volatile\n\nThin/volatile tokens: wider spreads, smaller order sizes\n\n**Best for:** Low-cap / noisy mints — survive adverse selection\n\n**How it works:** Wide spreads, small clips, higher stop_loss band, strict hard inventory cap.\n\n**Venues:** Meteora DLMM, Raydium CLMM / AMMv4, Pump.fun → Raydium/Meteora graduation pools, Jupiter (swap routing)\n\n**Key params:** `base_spread_bps`, `notional_per_trade_sol`, `stop_loss_pct`, `inventory_hard_cap_sol`\n\n### `skew_bull` — Bullish Skew MM\n\nMild uptrend: more aggressive bids to accumulate on dips\n\n**Best for:** Uptrend regimes — accumulate inventory on dips, lighter asks\n\n**How it works:** Elevated buy_dip + buy_into_flow; inventory_skew favors bids; higher soft inventory target.\n\n**Venues:** Meteora DLMM, Raydium CLMM / AMMv4, Jupiter (swap routing)\n\n**Key params:** `inventory_skew_factor`, `buy_dip`, `dip_trigger_pct`, `max_inventory_sol`\n\n### `skew_bear` — Bearish Skew MM\n\nDowntrend protection: heavier asks to reduce inventory\n\n**Best for:** Downtrend / dump risk — unload inventory, defensive quotes\n\n**How it works:** Elevated sell_into_flow;
